## 추가 내용

### 3.3.1 Persona 디자인 패턴

표 형식으로 4가지 패턴 정리:

| 패턴 | 설명 | 예시 |
|------|------|------|
| Role Prompting | "당신은 ~입니다" 역할 부여 | SOUL.md에서 성격 정의 |
| Two-Step Persona | 1단계 역할 설정, 2단계 작업 지시 | 시스템 프롬프트 + 사용자 메시지 분리 |
| Few-Shot Persona | 이상적 응답 예시 2-3개 포함 | 톤/형식 일관성 확보 |
| Context Up Front | 배경 정보를 프롬프트 앞에 배치 | RAG 결과를 컨텍스트로 주입 |

각 패턴별 OpenClaw 적용 설명 2-3줄:
- Role Prompting: SOUL.md + AGENTS.md에서 에이전트 성격 정의
- Two-Step: 시스템 프롬프트(역할)와 사용자 메시지(작업)를 분리
- Few-Shot: SOUL.md에 예시 대화를 포함하여 일관된 톤 유지
- Context Up Front: RAG 스킬 결과를 프롬프트 상단에 주입
