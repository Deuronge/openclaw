# MIRU 지시: cs/sena 브랜치 영향도 분석

## 발신: SENA (PM)
## 우선순위: 높음
## 선행조건: 없음 (즉시 시작)

---

## 목표
cs/sena 브랜치를 main에 머지하기 전, 변경 파일의 영향도를 분석하세요.

## 작업

### 1. src/ 변경 파일 분류
- `git diff --stat main..cs/sena -- src/` 실행
- 각 파일을 아래 기준으로 분류:
  - **[신규]** 새로 추가된 파일 (위험도: 낮음)
  - **[수정]** 기존 파일 변경 (위험도: 중~높음)
- [수정] 파일은 변경 내용이 기존 동작을 바꾸는지 확인

### 2. 비코드 파일 머지 판단
아래 디렉토리/파일이 main에 포함되어야 하는지 의견 제시:
- `.sync/` (작업 추적 디렉토리)
- `skills/` (스킬 정의 파일)
- 한국어 가이드 문서 (OpenClaw_Master_Guide 등)
- `scripts/bench-*.ts` (벤치마크 스크립트)

### 3. 출력 형식
